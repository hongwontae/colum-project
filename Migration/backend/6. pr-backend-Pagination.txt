1. 고려사항
1-1. 모든 Result을 가져와야 하나 vs login 사용자의 result만 가져와야 하나?
=> 모든 Result을 먼저 보여주고 login 사용자의 result 응답도 만들자 => 2개 만들기
=> JWT를 통해 유저 정보를 알 수 있고 관계 설정을 했기에 사용자의 result을 알 수 있다.

1-2. 모든 데이터를 한 페이지에 다 보여주는 것은 어리석은 짓
=> 페이지네이션 로직을 갖추자 10개 정도가 적당 or 8개

1-3. JWT를 통해 GET 요청을 보호? 너무 과한가? 
=> 그렇다면 다 보여주는것은 그냥 보여주고 일반 사용자의 게시물은 보호
